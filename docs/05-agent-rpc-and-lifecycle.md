# 05 — Agent RPC and Lifecycle

## Agent binary

- Binary name: `mux-agent`.
- Location on remote: `<home>/.mux/bin/mux-agent`.
- Lock file: `<home>/.mux/agent.lock` (contains PID and listen URL).
- Log file: `<home>/.mux/agent.log`.

## Agent startup

1. Check for stale lock: if `agent.lock` exists, check if PID is alive.
2. Stale lock (process dead): remove lock and any stale socket/port.
3. Held lock (process alive): agent is already running; return existing URLs.
4. Start agent process; it writes `agent.lock` with PID + URLs when ready.
5. Readiness poll: poll agent socket/port for up to N seconds.
6. Timeout: collect last 50 lines from `agent.log`; return `agent_start_timeout` error
   with log tail.
7. Concurrent-start safety: use lock-file atomicity (`O_CREAT | O_EXCL`).

## Agent listen URLs

- Streamlocal: `unix:<home>/.mux/agent.sock`.
- TCP: `tcp://127.0.0.1:<port>` (port chosen dynamically; persisted in lock file).

## RPC protocol

To be decided in mux-2n5. The following operations are required:

| Operation | Request | Response |
|---|---|---|
| `Health` | `{}` | `{ ok: bool }` |
| `CreateSession` | see below | `{ uuid, shortname, tmux_name }` |
| `ListSessions` | `{}` | `[{ uuid, shortname, tmux_name, status }]` |
| `GetSession` | `{ shortname }` | `{ uuid, shortname, tmux_name, status }` |
| `KillSession` | `{ uuid, repo_slug }` | `{ tmux_killed, workdir_removed }` |
| `Shutdown` | `{}` | `{}` |
| `StreamSessionEvents` | `{}` | unimplemented in v0.1; return UNIMPLEMENTED |

### CreateSession request fields

- `uuid`: pre-generated by client.
- `shortname`: pre-generated by client.
- `repo_slug`: normalised owner/repo.
- `branch`: git ref.
- `workdir_parent`: `<home>/.mux/<uuid>`.
- `repo_leaf`: last component of the repo path.

### KillSession effects

- `tmux_killed`: tmux session was found and killed.
- `workdir_removed`: workdir was removed (only for mux-created workdirs; see docs/02).

## Agent ownership

- Agent owns: creating tmux sessions, listing/getting tmux sessions, killing sessions.
- Agent does NOT read or write local SQLite.
- Agent maintains an in-memory `uuid → tmux_name` map for sessions it created.

## Error timeouts

- RPC request timeout: 30 seconds.
- Agent startup timeout: 60 seconds.
- Health probe interval during startup: 1 second.

## Shutdown drain

On `Shutdown` RPC: complete any in-flight `CreateSession` before exiting. Do not start
new sessions after Shutdown is received.
